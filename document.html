🚀 Шпаргалка по Git

Основные команды и базовые понятия

Три состояния файла: Working Directory → Staging (git add) → Repository (git commit)

⚙️ Настройка

КомандаОписание
git config --global user.name "Имя"Задать имя пользователя
git config --global user.email "mail@x"Задать email
git config --global init.defaultBranch mainВетка по умолчанию — main
git config --listПоказать все настройки

📁 Создание репозитория

КомандаОписание
git initСоздать новый репозиторий
git clone <url>Склонировать существующий

🔄 Основной цикл работы

КомандаОписание
git statusЧто изменилось
git add <file>Добавить файл в индекс
git add .Добавить все изменения
git commit -m "сообщение"Зафиксировать изменения
git logИстория коммитов
git log --oneline --graphКраткая история с графом

↩️ Отмена изменений

КомандаОписание
git restore <file>Откатить изменения в файле
git restore --staged <file>Убрать из индекса
git reset --soft HEAD~1Отменить коммит, оставить изменения
git reset --hard HEAD~1⚠️ Удалить последний коммит полностью
git revert <commit>Создать отменяющий коммит

🚑 Если что-то сломалось

СитуацияКоманда
Забыл что-то в последнем коммитеgit commit --amend
Кто менял строкуgit blame <file>
Потерял коммитgit reflog
Скачать только одну веткуgit clone -b <branch> <url>

📋 Типичный workflow (GitHub Flow)

  1. git switch -c feature/my-task — создать ветку
  2. Работаем, делаем коммиты
  3. git push -u origin feature/my-task
  4. Создаём Pull Request на GitHub
  5. После ревью — merge в main
  6. git switch main && git pull — обновляемся
